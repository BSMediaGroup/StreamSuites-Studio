# StreamSuites-Studio v0.2.1-alpha

**Release Date:** 2026-07-24
**Scope:** Browser Studio (independent product release with shared-platform compatibility)

## 🎯 Product version transition

`v0.2.1-alpha` introduces the first explicitly independent Studio product cycle number while retaining explicit shared-runtime compatibility to `0.5.4-alpha`.

- Runtime compatibility remains authoritative and is tracked as `v0.5.4-alpha`.
- Independent Studio product versioning is now active at `v0.2.1-alpha`.
- Runtime/Auth and shared state remain outside the Studio product boundary.

## 🧩 What is included in this cycle

- Release notes and metadata alignment to the new two-version model (product + shared).
- No Studio transport-layer media feature expansion claims are introduced in this pass.
- Existing Studio browser/media implementation posture is preserved unchanged for this release.

## 🛡️ Boundary and authority notes

- Studio continues as a browser-based client using existing Cloudflare media client behavior.
- Runtime/Auth owns rooms, invites, sessions, destinations, credentials, and shared state.
- Studio does not own shared-version authority; it consumes it via Runtime manifests and mirrors.

## 📄 Documentation artifacts

- Per-repo changelog added: `changelog/v0.2.1-CHANGELOG.md`.
- Links to compatibility and shared release context are included in this release file.

## ⚠️ Known limits / deferred work

- This cycle does not add new Studio broadcast, LiveKit, or Egress behavior.
- This release is metadata and release-structure synchronization for the new version model.
